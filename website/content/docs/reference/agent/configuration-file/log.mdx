# Log parameters for Consul agent configuration files

The page provides reference information for log parameters in a Consul agent configuration file.

## Log parameters

- `log_file` Equivalent to the [`-log-file` command-line flag](/consul/commands/agent#_log_file).

- `log_rotate_duration` Equivalent to the [`-log-rotate-duration` command-line flag](/consul/commands/agent#_log_rotate_duration).

- `log_rotate_bytes` Equivalent to the [`-log-rotate-bytes` command-line flag](/consul/commands/agent#_log_rotate_bytes).

- `log_rotate_max_files` Equivalent to the [`-log-rotate-max-files` command-line flag](/consul/commands/agent#_log_rotate_max_files).

- `log_level` Equivalent to the [`-log-level` command-line flag](/consul/commands/agent#_log_level).

- `log_json` Equivalent to the [`-log-json` command-line flag](/consul/commands/agent#_log_json).

- `enable_syslog` Equivalent to the [`-syslog` command-line flag](/consul/commands/agent#_syslog).

- `syslog_facility` When [`enable_syslog`](#enable_syslog)
  is provided, this controls to which facility messages are sent. By default, `LOCAL0` will be used.
